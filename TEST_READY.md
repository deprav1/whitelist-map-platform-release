# Безопасность и готовность E2E-тестов WhiteS (TEST_READY)

Этот файл содержит сводную информацию о готовности E2E-тестов для проекта WhiteS и подробную инструкцию по их запуску.

---

## 1. Сводка по E2E-тестам

Всего реализовано **104 E2E-теста**, распределенных по 4 файлам спецификаций:
1. **Tier 1: Feature Coverage** (`tests/e2e/tier1_feature_coverage.spec.js`) — **45 тестов**. Проверяют базовую работоспособность каждой из 9 UX/UI фич в стандартных условиях.
2. **Tier 2: Boundary Cases** (`tests/e2e/tier2_boundary_cases.spec.js`) — **45 тестов**. Проверяют устойчивость системы, обработку ошибок, граничные значения (например, длинные URL, блокировка LocalStorage, спам-клики, медленное 3G соединение) для всех 9 фич.
3. **Tier 3: Cross-Feature Interactions** (`tests/e2e/tier3_cross_feature.spec.js`) — **9 тестов**. Интеграционные тесты, проверяющие взаимное влияние нескольких фич (например, сохранение темной темы при работе в офлайне через SW, наложение эффектов стекла и тактильного масштаба).
4. **Tier 4: Real-World Workload** (`tests/e2e/tier4_real_world.spec.js`) — **5 тестов**. Эмулируют сквозные пользовательские сценарии (мобильный интерфейс на iPhone 12, офлайн-режим с отключенными тайлами, полная навигация клавиатурой с проверкой доступности WCAG AA, нагрузочный тест с 1000 записями и обмен состоянием через URL параметры).

---

## 2. Инструкция по запуску

Тесты запускаются с помощью встроенного Node.js веб-сервера (для обеспечения Secure Origin для Service Worker) и Playwright.

### Установка зависимостей
Если зависимости ещё не установлены, выполните:
```bash
npm install
```

### Запуск тестов
Для запуска всей тестовой инфраструктуры и прогона всех 104 тестов выполните:
```bash
npm test
```
*Этот скрипт запускает локальный сервер на порту `8080` и передает управление в `playwright test`.*

Для запуска тестов с графическим интерфейсом Playwright UI (удобно для отладки):
```bash
node tests/run_tests.js --ui
```

Для запуска конкретного файла тестов:
```bash
node tests/run_tests.js tests/e2e/tier1_feature_coverage.spec.js
```

Для запуска конкретного тест-кейса по названию:
```bash
node tests/run_tests.js -g "T1.1.1"
```

## 3. Ожидаемые результаты прогона

На текущем этапе проекта (Milestone 1) большинство тестов, связанных с интерактивными плашками, отправкой в Telegram, Service Worker и кастомными стилями, **будут падать**. Это **нормальное и ожидаемое поведение**, так как сами фичи планируется реализовать в рамках Milestones 2, 3 и 4. 

Все тесты написаны **честно** (без жестко захардкоженных фейковых результатов), они обращаются к реальным DOM-элементам, проверяют computed-стили и эмулируют реальный браузер. Тесты будут автоматически проходить (зеленеть) по мере реализации соответствующих UX/UI улучшений.
