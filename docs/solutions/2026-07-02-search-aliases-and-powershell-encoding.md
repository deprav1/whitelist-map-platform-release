# Решение проблем с поиском по синонимам и кодировкой скриптов PowerShell

* **Дата решения**: 2026-07-02
* **Тема**: Логика поиска по алиасам и кодировка скриптов PowerShell в Windows

---

## 1. Problem (Проблема)

1. Поиск по русскому слову *"телеграм"* или *"ютуб"* на карте не находил записи, в которых содержались только английские слова *"Telegram"* или *"YouTube"*. Кроме того, поиск по нескольким словам (например, *"Москва МТС"*) не работал, если слова в тексте отчета шли не подряд.
2. При запуске скриптов сборки пакета (`package-public-lite.ps1`) PowerShell падал с синтаксической ошибкой:
   ```text
   ParserError: TerminatorExpectedAtEndOfString
   ```

---

## 2. Root Cause (Причина)

1. **Баг поиска**: 
   - Функция `normalizeText` автоматически добавляла все синонимы к строке запроса, склеивая их через пробел.
   - Поиск выполнялся наивной проверкой подстроки: `normalizeText(haystack).includes(searchString)`. Если пользователь вводил *"телеграм"*, поисковый запрос превращался в *"телеграм telegram телега"*, и система пыталась найти эту длинную строку целиком внутри текста отчета.
   - Синонимы были односторонними (только с русского на английский), поэтому при поиске английского слова русские эквиваленты не искались.
2. **Баг кодировки**:
   - Скрипты PowerShell `.ps1` были сохранены в формате UTF-8 без сигнатуры BOM (Byte Order Mark).
   - Windows PowerShell (v5.1) по умолчанию считывает файлы без BOM в системной ANSI-кодировке (например, Windows-1251). В результате русские буквы считывались некорректно, байты искажались, что приводило к разрушению кавычек строк и синтаксическому сбою парсера.

---

## 3. Solution (Решение)

1. **Исправление поиска**:
   - Упрощена нормализация в `normalizeText` (теперь она только очищает текст от знаков препинания и приводит к нижнему регистру).
   - В `searchAliases` добавлены двусторонние соответствия (русский $\leftrightarrow$ английский).
   - В `getFilteredReports` поисковый запрос разбивается на слова (токены). Для каждого слова из `searchAliases` извлекаются синонимы. Отчет считается найденным, если для **каждого** слова из запроса нашлось совпадение (с учетом его синонимов) в тексте.
2. **Исправление кодировки**:
   - Файлы скриптов перекодированы в формат UTF-8 с BOM с помощью .NET метода:
     ```powershell
     [IO.File]::WriteAllText('путь_к_файлу', [IO.File]::ReadAllText('путь_к_файлу', [Text.Encoding]::UTF8), [Text.Encoding]::UTF8)
     ```

---

## 4. Prevention (Предотвращение)

1. **Правила поиска**: При реализации поиска по ключевым словам/тегам всегда разделять поисковую строку пользователя на отдельные термы (AND-поиск) и искать совпадения по каждому терму независимо.
2. **Правила кодировок**: Все скрипты PowerShell (`.ps1`), содержащие кириллические символы в строках или комментариях, обязательно должны сохраняться в кодировке **UTF-8 с BOM**, чтобы корректно выполняться на любых версиях Windows PowerShell.
