# Решение проблем настройки Playwright и запуска E2E-тестов (2026-07-02)

## 1. Проблема: Ошибка `MODULE_NOT_FOUND` для `@playwright/test`
* **Root Cause**: В корневом каталоге отсутствовал файл `package.json` и локальная папка `node_modules`, из-за чего `npx playwright test` не мог импортировать библиотеку `@playwright/test` из конфигурационного файла `playwright.config.js`.
* **Solution**: Создан файл `package.json` с зависимостью `"@playwright/test"`, и выполнена команда `npm install` для установки пакетов локально.
* **Prevention**: Всегда инициализировать `package.json` с зависимостями тестирования перед запуском сьюта.

## 2. Проблема: Отсутствие бинарных файлов Chromium (`browserType.launch: Executable doesn't exist`)
* **Root Cause**: Playwright требует наличия конкретных сборок браузеров в кэше пользователя. При первом запуске в изолированном окружении Chromium не был установлен.
* **Solution**: Запущена команда `npx playwright install chromium`, которая загрузила необходимые бинарные файлы в локальный кэш пользователя.
* **Prevention**: Добавлена инструкция по скачиванию браузеров (`npx playwright install`) в `TEST_READY.md` и `README` проекта.

## 3. Проблема: `SyntaxError: Invalid regular expression` в `tier3_cross_feature.spec.js`
* **Root Cause**: В регулярном выражении `/rgb\(5,\s*150,\s*9)/` была экранирована открывающая круглая скобка `\(`, но закрывающая круглая скобка `)` осталась неэкранированной, что привело к синтаксической ошибке регулярного выражения в V8.
* **Solution**: Символ `)` был корректно экранирован: `/rgb\(5,\s*150,\s*9\)/`.
* **Prevention**: Проводить проверку регулярных выражений с помощью линтера или юнит-тестов и выполнять локальный пробный запуск тестов сразу после написания.
