# 2026-07-03: Форсирование тёмной темы для согласованности с картой CartoDB Dark Matter

## Problem (Проблема)
При использовании светлой темы операционной системы/браузера стеклянные панели (Glassmorphism) окрашивались в полупрозрачный белый цвет. Накладываясь на угольно-черную карту `CartoDB Dark Matter`, это создавало мутный серо-молочный налёт ("Milky Effect"). Интерфейс выглядел грязным, имел низкую контрастность и терял ощущение премиального SaaS-продукта.

## Root Cause (Причина)
Стили приложения использовали медиа-запросы `@media (prefers-color-scheme: dark)` для адаптации под настройки пользователя. Однако карта Leaflet была жестко настроена на использование темных тайлов `CartoDB Dark Matter` независимо от темы ОС. Из-за этого возникал конфликт стилей: светлый UI накладывался на тёмную карту.

## Solution (Решение)
1. **Удаление медиа-запросов**: Из `styles.css` были удалены все медиа-запросы `@media (prefers-color-scheme: dark)`.
2. **Перенос тёмных переменных**: Все переменные тёмной темы (включая полупрозрачные RGBA цвета для стеклянных поверхностей) были перенесены непосредственно в блок `:root` как цвета по умолчанию.
3. **Мета-теги**: Мета-тег `color-scheme` в `index.html` был изменен со `light` на `dark`, а `theme-color` установлен в `#0f1117` для правильного отображения системных элементов браузера (например, окантовок окон и скроллбаров).

## Prevention (Предотвращение)
В картографических приложениях, использующих тёмные или специфические стилизованные тайлы, дизайн-система интерфейса должна быть жестко синхронизирована с цветовой гаммой карты. Если карта не имеет динамической смены светлой/тёмной подложки, интерфейс приложения должен принудительно работать в одной теме (Dark Only или Light Only).
