## Стратегический тезис: два движка, стиль — множитель

```
        ВСПЛЕСК ОТКЛЮЧЕНИЯ
        ↓                    ↓
   PUSH (виральность)   PULL (SEO)
   мессенджеры:         интент:
   каждый репост =      «нет интернета в X» →
   мини-инфографика     пререндер-страница региона
        ↓                    ↓
        └──→ КАРТА ←──────────┘
             ↓
   лёгкое действие (подтвердил/сообщил)
             ↓
        снова делится
```

- **Push** питается динамическим og:image + контекстными deep-link'ами. Backend (PHP на Timeweb) уже есть — технически возможно.
- **Pull** питается пререндер-страницами регионов из `reports.json` по крону — без переписывания SPA.
- **Стиль** — множитель обоих: в кризис шлют скриншоты, а по ссылкам судят по превью.
- **Приватность-first** — позиционный клин и差ерентор, а не издержка.

## Предпосылка: приватная аналитика

Без неё виральность и SEO нельзя измерить. Cookieless (self-hosted / Plausible).
События: `share_clicked`, `confirm_clicked`, `report_submitted`, `region_page_view`, `deeplink_open`.
Совместимо с красными линиями (без cookies/пикселей/PII).

## План по фазам

### Фаза A — «Скриншот-готовность» (стиль как множитель) · ~1.5 нед
- Единый индикатор доверия `trustScore` = свежесть × уверенность × подтверждения → «высокая/средняя/требует проверки»; stale приглушать. (P1.1)
- Живой герой-статус «Сейчас: N инцидентов в K регионах» — основа og:image и скриншота.
- Карта без тайлов не должна выглядеть сломанной: SVG-контур РФ + макрозоны + подпись «облегчённая карта». (P1.3)
- Мобильные quick-filters: 2×2 grid / fade-chevron, короткие лейблы. (P1.4)
- Фирменная сигнатура: единая цвето-кодировка тяжести + одна узнаваемая анимация.

### Фаза B — Push-движок (виральные петли) · ~2 нед
- `?report=<id>` и `?region=<slug>` deep-link с подсветкой маркера и авто-раскрытием карточки. (P1.5 / 2.1)
- Контекстный share на каждой карточке и хотспоте региона с точным текстом.
- ⭐ Динамический og:image (PHP, SVG→PNG, кэш) для главной, `?region=`, `?report=`. Флагманский рычаг. (2.2)
- Момент «поделись» сразу после подтверждения/отправки.

### Фаза C — Pull-движок (SEO intent capture) · ~2 нед
- Пререндер-страницы `/r/<region-slug>/` из `reports.json` по крону/PHP: текстовая сводка + ссылка в карту с фильтром. (P2.2)
- `WebPage`/`Dataset`/`ItemList` JSON-LD на региональных страницах, без `SpecialAnnouncement` как DoD-цели.
- sitemap: добавить региональные URL, убрать `reports.json`; перелинковка карта ↔ регионы.

### Фаза D — Усиление и удержание · ongoing
- Агрегация подтверждений из таблицы `confirmations` в `reports.json` (замыкает недавнюю фичу).
- Embed-виджет `/embed/?region=` для TG-каналов и локальных медиа + страница «Для СМИ». (2.4)
- Авто-пост в Telegram-канал при крупном инциденте с og-картинкой Фазы B. (3.3)
- Подписка на регион (web push через SW / TG-бот). (3.2)

## Метрики успеха

| Метрика | Цель |
|---|---|
| Доля визитов с шерингом (`share_clicked`/визиты) | > 5% |
| Превью ссылки с картинкой в TG/VK/WhatsApp | 100% основных |
| Индексируемые региональные страницы | все активные регионы |
| Органика по «нет интернета в <город>» | растущие показы (GSC) |
| Подтверждений на отчёт (медиана) | ≥ 3 |
| Первый полезный экран, 3G | < 2 с |

## Красные линии (не нарушать ради роста)

Без PII / точного GPS / регистрации; og:image и региональные страницы — только по агрегатам,
без авторов/адресов/сырых комментариев; никаких тяжёлых трекеров; список работает без внешних тайлов.
Приватность — часть бренда.

## Приоритет по ROI

Фаза A → B (особенно динамический og:image) → C. Аналитику включить до/параллельно A.

## Статус выполнения

- **Фаза A — выполнена и в проде** (v14+): индикатор доверия, 2×2 quick-filters, схематичная карта.
- **Фаза B — выполнена и в проде** (v17): `?report=`/`?region=` deep-links; отчёт-зависимый шеринг; динамический `api/og.php` (main/region/report, GD PNG, fallback на статику); `share.php` + rewrite отдаёт crawler'у контекстные OG-мета; нудж «Поделиться» после подтверждения.
- **Дальше — Фаза C (SEO intent capture):** сначала confirmation aggregation и privacy-safe analytics, затем пререндер-страницы `/r/<region>/` из `reports.json`, `WebPage`/`Dataset`/`ItemList` JSON-LD на них, расширение `sitemap.xml`.
- **Кросс-задача:** приватная аналитика (события `share_clicked`/`confirm_clicked`/…) — чтобы измерять эффект A/B.
