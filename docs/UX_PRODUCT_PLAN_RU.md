# UX-план Где белые списки?

Где белые списки? должен ощущаться как спокойный аварийный инструмент. Пользователь приходит не “изучать проект”, а быстро понять, что работает сейчас, насколько данным можно доверять и как безопасно добавить наблюдение.

## UX-принципы

- Сначала ответ, потом карта.
- На мобильном сначала список, потому что карта и тайлы тяжелее.
- Нет аккаунта, телефона, email и точной геолокации.
- Любая форма начинается с предупреждения о безопасности.
- У каждой отметки должны быть свежесть, уверенность и примерность места.
- Карта не должна быть единственным способом чтения данных.
- Цвет помогает, но не является единственным носителем смысла.

## Первый экран

P0:

- статус данных: демо/живые;
- время обновления;
- количество отчетов и свежих отметок;
- список последних отметок;
- CTA `Сообщить`;
- быстрые фильтры;
- пояснение, что публикуются только модерированные записи.

P1:

- “последнее подтверждение”;
- “восстановлено” как отдельный счетчик;
- активные фильтры над списком;
- короткая ссылка “О данных”.

Сделано в публичном прототипе:

- добавлен блок “Сейчас заметнее всего” с горячими регионами;
- добавлены кликабельные сводки по операторам;
- активные фильтры можно снимать по одному;
- CTA `Сообщить` доступен и в шапке, и в списке;
- форма сообщения отправляет наблюдение в премодерацию, а при недоступном endpoint оставляет безопасный черновик.
- добавлен вычисляемый блок “Инциденты” поверх опубликованных отметок.
- в `О данных` добавлены вкладки “Данные”, “Правила”, “Приватность”, “Безопасность”.
- кнопка `Пожаловаться` у карточки открывает жалобу на опасную, ошибочную, устаревшую или дублирующую запись с явной причиной.
- бейдж источника явно показывает `данные из кэша` или `резервные данные`, когда свежий публичный JSON недоступен.
- карточка отчета получила действие “Показать на карте”, которое на мобильном переключает из списка на карту и выделяет точку.
- первый экран перестроен в оперативную сводку: активные инциденты, свежие отметки, восстановления и всего опубликованных отметок.
- блоки “Инциденты”, “Сейчас заметнее всего” и “Операторы” объединены в один блок “Сводка”.
- форма сообщения получила обязательный safety-gate перед отправкой.
- блок “Ваша ситуация” уплотнен как быстрый ответ и получил контекстные действия “У меня тоже” и “Восстановилось”.
- состояния данных вынесены в отдельную панель с действиями для живых/демо-данных, кэша и резервного набора.
- пустые состояния различают “нет опубликованных данных” и “нет результатов по фильтрам”.
- добавлена low-bandwidth панель: список, фильтры, сводка и форма явно работают без внешних тайлов; из панели доступны `Открыть карту`, `Сообщить` и `Поделиться`.
- на мобильном первом экране Leaflet JS не грузится до открытия вкладки `Карта`; список остается быстрым и пригодным при слабой сети.
- на мобильном в списке добавлена нижняя sticky-панель: она появляется только при активных фильтрах, фильтры снимаются по одному или кнопкой `сбросить все`, а `Сообщить` остается заметным даже при прокрутке.
- добавлены локальные “мои места” без аккаунта: до пяти пар регион/оператор сохраняются только в браузере и применяются одним нажатием.
- добавлен безопасный share-dialog: он копирует нейтральный текст и URL текущих публичных фильтров, не включая локальные “мои места” или черновики.
- добавлена отдельная FAQ-страница для правил, приватности, жалоб и low-bandwidth режима.
- добавлены отдельные публичные страницы “Правила” и “Приватность” для шаринга и low-bandwidth чтения без открытия модального окна.
- добавлен read-only offline-shell: публичная оболочка, FAQ и fallback JSON открываются после первого визита даже при потере сети; прием наблюдений остается только online/pending.
- первый экран получил короткие trust-cues: без регистрации, премодерация, неофициальные отметки.
- форма сообщения получила inline-подсказку риска: телефон, адрес, IP, ссылки и другие рискованные паттерны подсвечиваются до отправки.

## Мобильный UX

Текущая политика:

- по умолчанию открывается `Список`;
- карта доступна отдельной вкладкой;
- Leaflet JS подгружается по требованию при открытии карты, а на десктопе стартует сразу;
- внешние тайлы выключены по умолчанию;
- пользователь может включить тайлы вручную;
- локальный кэш можно очистить.
- нижняя sticky-панель появляется только при активных фильтрах и держит CTA `Сообщить` под рукой в списке.
- “мои места” остаются локальными, без регистрации и точной геолокации.
- шеринг передает только публичные фильтры и не раскрывает локальные настройки пользователя.
- service worker кэширует публичную оболочку, FAQ, правила, приватность и fallback JSON, но не кэширует `api/`, `submissions/`, POST-запросы и внешние тайлы.

Следующее:

- уточнить тексты пустых состояний после появления реальных данных;

## Форма сообщения

Текущий статус: отправка в `api/observations.php` с премодерацией; безопасный черновик остается fallback-сценарием.

P0 для реальной формы:

- примерное место;
- оператор;
- тип сети;
- тип проблемы;
- проверенные сервисы;
- время проверки;
- уверенность;
- краткий комментарий;
- предупреждение о персональных данных;
- отправка только в премодерацию.

Запрещено просить:

- ФИО;
- телефон;
- email;
- точный адрес;
- точные GPS;
- место работы/учебы;
- аккаунты;
- скриншоты с личными данными.

## Карточка отчета

P0 элементы:

- город/район + оператор;
- текстовый статус: отключение, белый список, частично, восстановлено;
- время проверки;
- свежесть;
- уверенность;
- подтверждения;
- проверенные сервисы;
- безопасный summary.

P1:

- кнопки `подтверждаю`, `восстановилось`, `ошибка`;
- вычисляемый статус ситуации: `Активно`, `Разные отметки`, `Восстанавливается`, `Восстановлено`, `Устарело`;
- ссылка “показать на карте”;
- объяснение примерности точки.

## Пустые и ошибочные состояния

Нужны отдельные тексты:

- нет опубликованных данных;
- нет результатов по фильтрам;
- данные из кэша;
- карта без внешних тайлов;
- демо-данные;
- отправка отчета пока не подключена;
- ошибка загрузки JSON.

## Дизайн-компоненты

- `status-pill`: источник/режим данных.
- `quick-filter`: быстрый фильтр категории.
- `filter-chip`: активный фильтр.
- `hotspot-card`: быстрый переход к заметному региону.
- `operator-chip`: быстрый фильтр по оператору.
- `report-row`: карточка отчета.
- `tag`: уверенность, свежесть, сервисы.
- `modal`: безопасная форма/информация.
- `cluster-marker`: региональный кластер на карте.

## Следующие UX-задачи

P0:

- проверить empty/error состояния на первых реальных данных и добавить новые тексты, если пользователи путаются;

P1:

- проверить заметность CTA `Сообщить` на первых реальных данных и уточнить пустые состояния по фактическим вопросам пользователей;
- развить вычисляемые инциденты в отдельную серверную сущность, когда появятся реальные модерированные дубли;
- добавить историю восстановления;
- добавить региональные страницы;

P2:

- PWA/offline;
- виртуализация списка;
- плотностная кластеризация;
- экспорт для исследователей.
