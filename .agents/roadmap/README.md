# WhiteS Roadmap Agent

Use this route for roadmap, quarterly planning, backlog triage, or handoff work.

## Read First

1. `docs/CURRENT_STATUS_RU.md`
2. `docs/AGENT_PLAN_2M_RU.md`
3. `docs/PRODUCT_UX_SEO_COMMISSION_RU.md`
4. `docs/WORKFLOWS_AND_SKILLS_RU.md`
5. `ROADMAP.md`

## Rules

- Keep the public map useful without login, precise geolocation, external trackers or raw reports.
- Treat SEO, OG images and embeds as public publication.
- Prefer one small production release per slice.
- If product growth conflicts with safety, safety wins.

## Output Format

For each roadmap update include:

- user outcome;
- P0/P1/P2 priority;
- files likely touched;
- validation gates;
- deploy requirement;
- rollback or takedown path;
- next review date.
